## 📜 Descrição

A Global Solution 2026.1 desafia grupos a desenvolverem uma Prova de Conceito (POC) que responda: "Como a tecnologia espacial pode ser utilizada para melhorar a vida das pessoas, tornar processos mais eficientes e criar novas oportunidades na Terra ou fora dela?"

Nossa solução explora o potencial da economia espacial aplicada a um problema real, integrando programação Python, análise de dados, Inteligência Artificial, sensores e automação. O projeto demonstra como dados espaciais — seja de satélites, sensores remotos ou simulações — podem gerar insights acionáveis para áreas como monitoramento ambiental, agricultura inteligente, comunicação via satélite ou gestão de ambientes extremos.

A POC foi construída com base nos conhecimentos das Fases 3 e 4 do curso, utilizando Python com Pandas para manipulação de dados, Matplotlib/Seaborn para visualização, conceitos de Machine Learning introdutório para predição e, quando aplicável, ESP32 para coleta de dados via sensores. Todo o desenvolvimento foi documentado, versionado e apresentado em vídeo demonstrativo.

## 📁 Estrutura de pastas

Dentre os arquivos e pastas presentes na raiz do projeto, definem-se:

- <b>docs</b>: Documentação textual — brainstorm, atas de reunião, arquitetura da solução, storyboard, prints e diagramas. Também contém o PDF final de entrega.

- <b>src</b>: Código fonte desenvolvido em Python (scripts, notebooks, APIs), modelos de ML, inferências e, quando aplicável, códigos para ESP32/Arduino.

- <b>data</b>: Bases de dados utilizadas — CSV, JSON, Excel ou bases sintéticas geradas para a POC.

- <b>assets</b>: Imagens, logos, screenshots do dashboard e gráficos gerados.

- <b>README.md</b>: Arquivo guia e explicação geral sobre o projeto.

## 🔧 Como executar o código

### Pré-requisitos
- Python 3.10+
- pip

### Passo a passo

1. Clone o repositório:
```bash
git clone https://github.com/zjpza/fiap-gs-1.git
cd fiap-gs-1
```

2. Instale as dependências:
```bash
pip install -r requirements.txt
```

3. Execute os scripts na ordem indicada na documentação de cada pasta `src/`.

4. Para visualizações e notebooks:
```bash
jupyter notebook src/
```
